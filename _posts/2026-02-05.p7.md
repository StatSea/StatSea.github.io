## 프로젝트 개요

### 과제 제목
**야구 입문자를 위한 AI 해설위원**

---

## 과제 요약 (200자 이내)

야구 입문자가 겪는 규정과 응원 문화의 진입장벽을 낮추기 위한 **실시간 AI 해설 서비스**입니다.  
경기 맥락을 반영한 API 기반 구조를 통해 기존 정보 검색의 한계와 AI의 할루시네이션 문제를 동시에 해결하고, 사용자가 불필요한 검색 없이 상황에 맞는 정확한 설명과 응원 가이드를 받을 수 있도록 설계했습니다.

---

## 주제 선정 이유

야구는 규정이 복잡하고 응원 문화가 방대해 입문자에게 진입장벽이 높은 스포츠입니다.  
기존 검색이나 AI 서비스는 실시간 경기 맥락을 반영하지 못하거나 부정확한 정보를 제공하는 경우가 많았습니다.  
이에 **실시간 API 데이터와 프롬프트 설계**를 결합하는 방식이 가장 정확하고 효율적인 해결책이라 판단하여 본 주제를 선택했습니다.

---

## 타겟 사용자

- KBO 야구에 익숙하지 않은 관람객
- 처음 야구장을 방문한 직관 입문자
- 규칙 이해가 부족한 라이트 팬
- 중계는 보지만 상황 해석이 어려운 시청자
- KBO 규칙·응원 문화에 익숙하지 않은 팬 전반

---

## 시행착오 및 개선 과정

### 1. 가상 실시간 환경 구현
KBO 비시즌으로 실시간 데이터를 확보할 수 없는 한계를 해결하기 위해,  
**네이버 스포츠 과거 경기 데이터를 크롤링**하여 10초 단위로 순차 갱신하는 시나리오 주입 방식의 가상 중계 환경을 구현했습니다.

### 2. 규정 데이터 구조화 및 전용 API 개발
LLM이 복잡한 야구 규정을 임의로 해석하는 문제를 해결하기 위해  
KBO 규정집을 데이터프레임으로 전처리하고 중계 용어와 매핑하여 **규정 조회 전용 API**를 자체 개발했습니다.

### 3. 응원가 DB 구축 및 Fallback 로직
공식 API가 존재하지 않는 응원가 문제를 해결하기 위해  
팀·선수별 응원가 DB를 직접 구축했으며,  
데이터가 없는 경우 응원가를 생성하지 않고 **팀별 응원가 링크 안내**로 대체하도록 프롬프트를 수정했습니다.

### 4. 상황 인식 API 세분화
초기에는 경기 상황 인식이 부족했으나,  
경기 요약 / 라인업 / 투수·타자 정보 / 현재 상태 / 최근 이벤트를 **목적별 API**로 분리하여  
LLM이 루상 상황과 투타 구도를 정확히 인식하도록 개선했습니다.

### 5. 로컬 → 배포 환경 확장
로컬 환경에서 충분한 검증을 거친 뒤,  
배포 환경으로 확장하여 실제 서비스 형태로 접근 가능하도록 구현했습니다.

---

## 프롬프톤 참여 소감

이번 프로젝트를 통해 아이디어를 실제 서비스로 구현하는 전 과정을 경험할 수 있었습니다.  
단순한 챗봇이 아닌, **데이터 → API → 프롬프트 → 사용자 응답**으로 이어지는 구조를 설계하는 과정이 특히 인상 깊었습니다.

LaaS 플랫폼을 활용하면서 API 연동과 프롬프트 테스트를 빠르게 반복할 수 있었고,  
AI를 단순한 대화 도구가 아닌 **규칙을 가진 시스템**으로 설계해야 한다는 점을 깊이 체감했습니다.

팀원들과의 협업을 통해 각자의 강점을 살려 문제를 해결해 나갈 수 있었고,  
이번 경험을 계기로 AI 기반 서비스 구조 설계에 대해 더 깊이 고민해보고 싶다는 목표가 생겼습니다.
